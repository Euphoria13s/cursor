# Руководство пользователя: система складского учёта

## 1. Назначение

Приложение предназначено для учёта номенклатуры и движений товара на складе:
- создание категорий и товаров;
- фиксация поступлений;
- фиксация списаний;
- контроль остатков.

## 2. Вход в систему

1. Откройте страницу входа (`/login`).
2. Введите имя пользователя и пароль.
3. Нажмите **«Войти»**.

Если учётной записи нет:
1. Перейдите на `/register`.
2. Заполните форму регистрации.
3. После регистрации войдите в систему.

## 3. Навигация

После входа доступны разделы:
- **Dashboard** — общая сводка;
- **Categories** — список категорий;
- **Products** — список товаров;
- **Receipts** — документы поступления;
- **Writeoffs** — документы списания.

## 4. Работа с категориями

### Создать категорию
1. Откройте **Categories**.
2. Нажмите «Создать».
3. Заполните название и описание.
4. Сохраните.

### Изменить/удалить категорию
- В карточке категории выберите **Редактировать** или **Удалить**.

## 5. Работа с товарами

### Создать товар
1. Откройте **Products** → «Создать».
2. Заполните обязательные поля:
   - Наименование
   - SKU (уникальный артикул)
   - Категория
   - Ед. измерения
3. При необходимости заполните цены и минимальный остаток.
4. Нажмите «Сохранить».

### Что важно по остаткам
- `current_stock` — текущий остаток.
- `min_stock` — минимально допустимый запас.
- Контролируйте товары, где остаток приближается к `min_stock`.

## 6. Поступления (Receipts)

### Создать документ поступления
1. Откройте **Receipts** → «Создать».
2. Укажите номер документа, дату и поставщика.
3. Добавьте позиции документа (товар, количество, цена).
4. Сохраните документ.

### Статусы документа
- `draft` — черновик;
- `posted` — проведён;
- `cancelled` — отменён.

## 7. Списания (Writeoffs)

### Создать документ списания
1. Откройте **Writeoffs** → «Создать».
2. Укажите номер, дату, причину списания.
3. Добавьте позиции списания (товар, количество, цена).
4. Сохраните.

Рекомендуется выполнять списание только после проверки фактических остатков.

## 8. Поиск, фильтры и сортировка

В списках доступны:
- поиск по ключевым полям (например, наименование/SKU/номер документа);
- фильтрация по параметрам;
- сортировка по колонкам.

Это помогает быстро находить нужные товары и документы.

## 9. Типовые ошибки

- **401 Unauthorized**: выполните повторный вход.
- **400 Bad Request**: проверьте обязательные поля (например, SKU/номер документа).
- Невозможно выбрать чужую категорию/товар: данные изолированы по пользователям.

## 10. Рекомендации по эксплуатации

- Ведите единый формат SKU.
- Регулярно проверяйте товары с низким остатком.
- Не удаляйте документы без необходимости — лучше переводить в `cancelled`.
